## ūüéģ About the Game

**Jungle Warfare** is a multiplayer survival war game set in dense, hostile jungle. Think Rust ‚ÄĒ but deeper in the canopy, and with an RTS twist.

- ūü™Ķ **Build like Rust** ‚ÄĒ gather resources, construct and fortify your base in the jungle
- ūü™Ė **Command troops** ‚ÄĒ direct AI soldiers to raid, defend, and flank enemy positions
- ‚öĒÔłŹ **Raid other players** ‚ÄĒ knock down their bases, take their resources, dominate the map
- ūüĆŅ **The jungle is the third enemy** ‚ÄĒ terrain, wildlife, and visibility all work against you
- ūüĖ•ÔłŹ **Dedicated servers only** ‚ÄĒ no peer-to-peer, no local hosting, no offline play
- ūü•Ĺ **VR support** ‚ÄĒ a full VR client is available alongside the standard client
- ūüöę **Zero tolerance for cheating** ‚ÄĒ see the server rules below

> There is no save system. What you build, you defend. What you lose, you lose.

---

## ūüĖ•ÔłŹ Dedicated Servers & Anti-Cheat

Jungle Warfare runs exclusively on dedicated servers. There are no peer-to-peer options.

**How mods work in multiplayer:**  
Mods are installed on the server, not the client. When a player connects, the server automatically sends all active mods to their client. Players never need to manually download or install mods ‚ÄĒ it just works.

**Server rules are strictly enforced by the Jungle Warfare Dev Team:**

- Cheating, exploiting, or using modified clients to gain an unfair advantage results in a **permanent ban**
- Mods are distributed server-to-client automatically and run in a sandboxed environment ‚ÄĒ clients cannot inject unauthorised code
- Mods cannot access internals outside the published mod API

> ‚ö†ÔłŹ **Mods are for server customisation, not client advantage.**  
> If your mod gives one player an edge that others can't see or counter, it won't be allowed on any official server.

### Installing the Game

1. Download the client for your platform from [Releases](../../releases/latest)
2. Run the installer (Windows) or extract the archive (Linux/macOS)
3. Launch Jungle Warfare and connect to a server ‚ÄĒ no account or setup required

> There is no offline mode and no save system. Progress lives on the server.

### Setting Up a Dedicated Server

1. Download the dedicated server package for your platform from [Releases](../../releases/latest)
2. Extract the archive and run `JungleWarfare-Server`
3. Edit `server.cfg` to set your server name, player limit, and other options
4. Drop any `.pck` mod files into the `mods/` folder before starting
5. Forward the required ports and share your server address with players

Players connecting to your server will **automatically receive all server-side mods** ‚ÄĒ they do not need to install anything manually. See the [Modding](#ÔłŹ-modding) section for how to build mods.

## ūüõ†ÔłŹ Modding

Jungle Warfare's modding framework is built on **Godot 4** and **GDScript**. Mods are packed into Godot `.pck` files, installed on the server, and automatically delivered to players when they connect. You do **not** need the game source code ‚ÄĒ the Mod SDK is self-contained.

### Prerequisites

| Tool | Why you need it |
|---|---|
| **Python 3.x** | To run `build_mod.py` (packer) and `read_pck.py` (inspector) |
| **Godot 4.x** | To edit and preview mod scenes and scripts ‚ÄĒ not required, but strongly recommended |
| A code editor | Any editor works ‚ÄĒ VS Code with the GDScript extension is a good choice |

---

### Mod Folder Structure

```
my_mod/
‚Ēú‚ĒÄ‚ĒÄ manifest.json          ‚Üź Required: mod metadata (id, name, version, description)
‚Ēú‚ĒÄ‚ĒÄ mod_main.gd            ‚Üź Entry point ‚ÄĒ runs when your mod loads on the server
‚Ēú‚ĒÄ‚ĒÄ tools/
‚Ēā   ‚ĒĒ‚ĒÄ‚ĒÄ my_tool.gd         ‚Üź Custom tools (extend the base tool class)
‚Ēú‚ĒÄ‚ĒÄ entities/
‚Ēā   ‚ĒĒ‚ĒÄ‚ĒÄ my_entity.tscn     ‚Üź Spawnable entities (Godot scenes)
‚Ēú‚ĒÄ‚ĒÄ sounds/
‚Ēā   ‚ĒĒ‚ĒÄ‚ĒÄ my_sound.ogg       ‚Üź Audio assets
‚ĒĒ‚ĒÄ‚ĒÄ icons/
    ‚ĒĒ‚ĒÄ‚ĒÄ my_tool.png        ‚Üź Icons shown in server UI / HUD
```

When packed, all assets are available at runtime under `res://mods/<your_mod_id>/`.

---

### Naming Rules

Mod IDs and **all filenames** must use lowercase letters, numbers, and underscores only:

```
my_jungle_mod   ‚úď
MyJungleMod     ‚úó  (no capitals)
my-jungle-mod   ‚úó  (no hyphens)
```

---

### manifest.json

```json
{
  "id":          "my_mod",
  "name":        "My Mod",
  "version":     "1.0.0",
  "author":      "YourName",
  "description": "What does your mod do?"
}
```

The `id` field must exactly match your mod's folder name. `author` is optional but encouraged.

---

### Supported Asset Types

| Type | Formats | Folder |
|---|---|---|
| Scripts | `.gd` | Anywhere in the mod folder |
| Scenes / entities | `.tscn` | `entities/` |
| Audio | `.ogg` | `sounds/` |
| Icons / textures | `.png` | `icons/`, `textures/` |

Other formats (`.wav`, `.jpg`, `.tres`, `.res`) may work but are untested ‚ÄĒ stick to the list above.

---

### Avoiding Conflicts with Other Mods

Always prefix your registered IDs with your mod ID:

```gdscript
ToolRegistry.register("my_mod:my_tool", ...)    # ‚úď good
ToolRegistry.register("my_tool", ...)           # ‚úó bad ‚ÄĒ may clash with another mod
```

This applies to `ToolRegistry`, `EntityRegistry`, `Console` commands, and any global keys.

---

### Available Autoloads

These are always available in your GDScript:

| Autoload | What it does |
|---|---|
| `ModLoader` | Check if other mods are loaded |
| `ToolRegistry` | Register and look up tools |
| `EntityRegistry` | Register and look up spawnable entities |
| `GameEvents` | Connect to game-wide signals |
| `Console` | Register commands, print to the in-game console |
| `HUD` | Show hints, crosshairs, and notifications |
| `Audio` | Play sounds in 2D or 3D space |

### GameEvents Signals

| Signal | When it fires |
|---|---|
| `player_joined(player)` | A player joins the server |
| `player_left(player)` | A player disconnects |
| `map_changed(map_id)` | The map changes |

### Building Your Mod

Run the packer from the SDK folder:

```
python build_mod.py
```

You'll be prompted for:

```
Mod name:         my_mod
Source directory: C:\projects\my_mod    ‚Üź the folder containing manifest.json
Output directory: ./dist                ‚Üź where to save the .pck
Godot version:    4.2
```

To verify the output before deploying:

```
python read_pck.py
```

All paths inside the `.pck` should start with `res://mods/<your_mod_id>/`. If they don't, you pointed the builder at the wrong folder.

---

### Installing a Mod on a Server

Drop the `.pck` into the server's mods folder:

- **Windows:** `%APPDATA%\JungleWarfare\mods\`
- **Linux / macOS:** `~/.local/share/JungleWarfare/mods/`

Restart the server. Connecting clients will receive the mod automatically ‚ÄĒ no action needed on the player's side. The `[ModLoader]` lines in the server log will confirm whether the mod loaded successfully.

## ūüźõ Troubleshooting

**Mod not recognised by the server**
- `manifest.json` must be at the **root** of your mod folder, not inside a subfolder
- Required fields: `id`, `name`, `version`, `description` ‚ÄĒ all must be present
- The `id` field must exactly match your folder name, lowercase and underscores only
- Double-check the `.pck` is in the correct mods folder for your OS (see above)

**Assets not loading (missing textures, sounds, scenes)**
- Run `python read_pck.py` ‚ÄĒ paths must start with `res://mods/<your_mod_id>/`
- Make sure you pointed `build_mod.py` at the folder that *contains* `manifest.json`, not a parent folder
- Filenames are case-sensitive on Linux ‚ÄĒ use lowercase everywhere

**Script errors or mod crashes on load**
- Confirm `mod_main.gd` extends the correct base class ‚ÄĒ check the tutorial mod for the signature
- If you extended a framework class, make sure the `.pck` was built against a matching Godot version; rerun `build_mod.py` and enter the correct version when prompted
- Check the server log for `[ModLoader]` lines ‚ÄĒ they will describe exactly what failed

**`build_mod.py` fails or outputs an empty `.pck`**
- Confirm you are running Python 3 (`python3 --version`)
- The source directory must contain `manifest.json` at its root
- On Windows, use a full absolute path with no trailing slash (e.g. `C:\projects\my_mod`)

**ID conflicts with another mod**
- Prefix all registered IDs with your mod ID (see above)
- Use `ModLoader.is_mod_loaded("other_mod_id")` to check before registering shared resources

## ūüĒĘ Versioning

Jungle Warfare releases follow **Semantic Versioning** ‚ÄĒ every release is numbered `vMAJOR.MINOR.PATCH`.

| Number | Changes when... | Example |
|---|---|---|
| **MAJOR** | A huge update ‚ÄĒ new core game systems, or breaking changes to the mod API | `v1.0.0` ‚Üí `v2.0.0` |
| **MINOR** | New features, new content, or new mod API additions that don't break existing mods | `v1.3.0` ‚Üí `v1.4.0` |
| **PATCH** | Bug fixes and hotfixes only ‚ÄĒ nothing new, nothing broken | `v1.4.1` ‚Üí `v1.4.2` |

**Rules:** when MAJOR increases, MINOR and PATCH reset to `0`. When MINOR increases, PATCH resets to `0`.

> Releases marked `v0.x.x` are **early access** ‚ÄĒ the game is still in active development. The `v1.0.0` milestone marks full launch.

If you are a modder, pay attention to MAJOR bumps ‚ÄĒ these may require updates to your mod to stay compatible. MINOR and PATCH releases will not break existing mods.
